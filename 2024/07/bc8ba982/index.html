巴雷特模乘

公式/原理：

a mod b=a−⌊ax264⌋bx=264b\begin{aligned}a\bmod b&=a-\left\lfloor\dfrac{ax}{2^{64}}\right\rfloor b\\x&=\dfrac{2^{64}}{b}\end{aligned} amodbx​=a−⌊264ax​⌋b=b264​​

注意 2642^{64}264 会爆 unsigned long long ，要用 __int128_t 。

当 a<ba<ba<b 时就不用了。

稳定排序指排序前后关键字大小相同的元素的相对位置不变（原来谁在前现在谁在前）。

计数排序

演示网站；演示网站（上方选择 COU）。

思路：

  1. 统计每个值出现几次。
  2. 求出出现次数的前缀和，我称之为计数数组。
  3. 对于每一个元素根据计数数组对应下标的值丢进另一个数组的相应位置，然后计数数组减一。
    • 应该倒着枚举，使得计数排序成为稳定的排序。

我的理解：计数数组存储的是不大于自己的元素的个数；减一是为了处理多个元素相同的情况。

基数排序

思路：把每个数据拆成多个关键字，根据优先级倒着枚举每个关键字跑稳定排序。

例：将 323232 位整数拆成 444 段，每段 888 位，然后跑四次计数排序。
因为原数相当于以第一段为第一关键字、第二段为第二关键字等排序，故先对第四段排序，再对第三段排序，以此类推。

线段树分治

虽然名字里有线段树，但是这玩意没用到线段树，只用到线段树的思想。

问题：有一些仅作用于一段时间的修改和一些不同时刻的询问，离线。

思路：在时间上“建立”一棵线段树，把每个修改拆成一些小线段，完整覆盖了某条线段时打一个标记。接着一遍 dfs，进入时修改，退出时回溯。

宏定义

#define 中的 #x 代表将参数 x 的变量名取出来，形式为字符串常量。

下面是一些 C++ 编译器内置的预定义宏，它们不能被取消定义（#undef），也不能被编程人员重新定义。

  • __DATE__ 代表日期，是形式为 Mmm dd yyyy 的字符串常量（例：Jul 3 2024）。
  • __TIME__ 代表时间，是形式为 hh:mm:ss 的字符串型常量（例：19:51:24）。
  • __FILE__ 表示当前源文件名，类型为字符串常量（例：Test.cpp）。
  • __func__ 和 __FUNCTION__ 都代表当前函数的函数名，类型为字符串常量（例：solve）。
  • __LINE__ 代表当前程序行的行号，类型为十进制整数常量（例：28）。
